03
Memory Management
Leaks kill battery and cause crashes ‚Äî most common cause of ANR and OOM errors
ü§ñ
Android
Android
Use LeakCanary in debug builds ‚Äî auto-detects and reports Activity/Fragment leaks
Never hold Context reference in static fields ‚Äî causes Activity leaks
Use WeakReference when callbacks hold Activity refs across async operations
Bitmap management ‚Äî use Glide/Coil with built-in lifecycle-aware caching
Cancel Coroutines in onDestroy / viewModelScope auto-cancels on clear
üçé
iOS
iOS
Use [weak self] in closures ‚Äî prevents retain cycles with self
Instruments Leaks tool ‚Äî run regularly to catch abandoned memory & cycles
Avoid strong delegate references ‚Äî always declare delegates as weak var
NSCache over NSDictionary ‚Äî auto-evicts under memory pressure
Use autoreleasepool in tight loops processing many objects
üíô
Flutter
Flutter
Always dispose controllers ‚Äî AnimationController, TextEditingController, StreamSubscription
Use StreamSubscription.cancel() in dispose() to avoid stream leaks
Image.network caches in memory ‚Äî use CachedNetworkImage with disk cache for large sets
DevTools Memory tab ‚Äî take heap snapshots before/after user flows to spot regressions
‚öõÔ∏è
React Native
React Native
Remove event listeners in useEffect cleanup ‚Äî DeviceEventEmitter, AppState, etc.
Avoid large objects in state/redux ‚Äî store IDs, not full objects in component state
Use react-native-fast-image over Image for aggressive memory-aware caching
Clear timers ‚Äî always clearTimeout/clearInterval in useEffect cleanup

Network & Data
Latency is the enemy ‚Äî minimize requests, maximize cache, compress everything
üì°
Request Strategy
All
HTTP/2 multiplexing ‚Äî single connection for multiple requests
Pagination ‚Äî cursor-based over offset. Load 20 items, not 200
Debounce search inputs (300ms) ‚Äî avoid request per keystroke
Cancel inflight requests when screen leaves ‚Äî use CancelToken/AbortController
Batch API calls ‚Äî GraphQL or composite endpoints over 10 REST calls
üíæ
Caching Strategy
All
Stale-while-revalidate ‚Äî show cached data immediately, refresh in background
ETag / Last-Modified headers ‚Äî 304 Not Modified saves full payload download
Android: OkHttp Cache ‚Äî built-in disk cache with HTTP cache semantics
iOS: URLCache ‚Äî configure memory + disk capacity appropriately
Use offline-first ‚Äî SQLite/Room/CoreData as source of truth, sync to server
üóúÔ∏è
Payload Optimization
All
gzip / brotli compression on all JSON ‚Äî 70-80% size reduction
Protobuf / MessagePack over JSON for high-frequency data
Images: WebP format ‚Äî 25-34% smaller than JPEG at same quality
Request only needed fields ‚Äî GraphQL field selection or sparse fieldsets
CDN for static assets ‚Äî serve images from edge close to user

App Startup Time
Cold start <2s, warm start <1s ‚Äî first impressions decide if users stay
ü§ñ
Android
Android
Use App Startup library ‚Äî lazy-initialize components, not all at once
Baseline Profiles ‚Äî pre-compile hot paths, 30-40% faster cold start
Avoid work in Application.onCreate ‚Äî defer until actually needed
SplashScreen API ‚Äî instant splash, no blank white flash on launch
üçé
iOS
iOS
Move work out of applicationDidFinishLaunching ‚Äî target <400ms
Use lazy var for expensive singletons ‚Äî init on first use, not app start
Reduce dylib count ‚Äî merge frameworks, fewer dynamic libs = faster launch
Background fetch pre-warms data ‚Äî content ready before user opens app
üíô
Flutter
Flutter
Deferred components ‚Äî split app into loadable modules, download on demand
Use flutter build with --tree-shake-icons ‚Äî removes unused icon glyphs
Impeller engine ‚Äî pre-compiled shaders, no startup jank on first render
Minimize plugin initializations in main() ‚Äî use async init patterns

React Native
React Native
Hermes ‚Äî pre-compiled bytecode ships in bundle, faster JS parse
RAM Bundles ‚Äî load only required modules at startup, rest on demand
Lazy-load heavy screens ‚Äî React.lazy + Suspense for non-initial routes
Reduce native module init ‚Äî only auto-link modules you actually use

Android Battery
Android
Use WorkManager for background tasks ‚Äî battery-aware, respects Doze mode
JobScheduler constraints ‚Äî run sync only on WiFi + charging when possible
Batch network requests ‚Äî 1 wake-up for 10 requests vs 10 individual wake-ups
Use Battery Historian tool ‚Äî visualize wakelocks, alarms, sync frequency
Avoid partial wakelocks ‚Äî use FCM push to wake app, not polling
üîã

iOS

Background App Refresh ‚Äî request time wisely, system may deny in low-power mode
Use signpost intervals in Instruments Energy log to find expensive operations
Avoid CLLocationManager with high accuracy when approximate is sufficient
Use URLSession background tasks for large uploads/downloads
